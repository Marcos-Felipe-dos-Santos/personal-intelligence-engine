# PIE ŌĆö Data Model

## Tables

### `raw_entries`

Stores the original, unprocessed text input exactly as captured.

| Column | Type | Description |
|--------|------|-------------|
| `id` | TEXT PK | UUID string |
| `content` | TEXT NOT NULL | Original text content |
| `source` | TEXT NOT NULL | Origin identifier (e.g., "cli", "api") |
| `status` | TEXT NOT NULL | Pipeline status: pending, processed, needs_review, error |
| `created_at` | TEXT NOT NULL | ISO 8601 UTC timestamp |
| `updated_at` | TEXT NOT NULL | ISO 8601 UTC timestamp |
| `metadata_json` | TEXT | Optional JSON metadata |
| `content_hash` | TEXT NOT NULL | SHA-256 hash of content |

### `structured_entries`

Stores the extracted, validated, structured representation of a raw entry.

| Column | Type | Description |
|--------|------|-------------|
| `id` | TEXT PK | UUID string |
| `raw_entry_id` | TEXT NOT NULL FK | Reference to raw_entries.id |
| `entry_type` | TEXT NOT NULL | Taxonomy type (see Entry Types below) |
| `project` | TEXT | Optional project association |
| `summary` | TEXT NOT NULL | Human-readable summary |
| `confidence` | REAL NOT NULL | Extraction confidence (0.0 ŌĆō 1.0) |
| `structured_json` | TEXT NOT NULL | Full extraction result as JSON |
| `validation_status` | TEXT NOT NULL | valid, needs_review, or invalid |
| `created_at` | TEXT NOT NULL | ISO 8601 UTC timestamp |
| `updated_at` | TEXT NOT NULL | ISO 8601 UTC timestamp |

### `audit_logs`

Immutable log of all pipeline actions for traceability.

| Column | Type | Description |
|--------|------|-------------|
| `id` | TEXT PK | UUID string |
| `raw_entry_id` | TEXT | Optional reference to raw_entries.id |
| `action` | TEXT NOT NULL | Action type (see Audit Actions below) |
| `actor` | TEXT NOT NULL | Who performed the action (default: "system") |
| `method` | TEXT | Method used (e.g., "fake_extractor") |
| `model_name` | TEXT | Model name if LLM was used |
| `prompt_version` | TEXT | Prompt version if applicable |
| `input_hash` | TEXT | Hash of the input data |
| `output_hash` | TEXT | Hash of the output data |
| `status` | TEXT NOT NULL | success, warning, or error |
| `error_message` | TEXT | Error details if status is error |
| `created_at` | TEXT NOT NULL | ISO 8601 UTC timestamp |

### `generated_files`

Tracks files generated by PIE (Markdown notes, reports, etc.).

| Column | Type | Description |
|--------|------|-------------|
| `id` | TEXT PK | UUID string |
| `raw_entry_id` | TEXT | Optional reference to raw_entries.id |
| `file_type` | TEXT NOT NULL | Type of file (e.g., "markdown_note") |
| `path` | TEXT NOT NULL | File system path |
| `content_hash` | TEXT | SHA-256 hash of file content |
| `created_at` | TEXT NOT NULL | ISO 8601 UTC timestamp |

### `reports`

Stores report metadata and references.

| Column | Type | Description |
|--------|------|-------------|
| `id` | TEXT PK | UUID string |
| `report_type` | TEXT NOT NULL | Report type (e.g., "daily") |
| `date_start` | TEXT NOT NULL | Report period start (YYYY-MM-DD) |
| `date_end` | TEXT NOT NULL | Report period end (YYYY-MM-DD) |
| `summary` | TEXT NOT NULL | Text summary of the report |
| `file_path` | TEXT | Path to the generated report file |
| `source_entry_ids_json` | TEXT NOT NULL | JSON array of structured entry IDs |
| `created_at` | TEXT NOT NULL | ISO 8601 UTC timestamp |

### `schema_migrations`

Tracks applied database migrations.

| Column | Type | Description |
|--------|------|-------------|
| `version` | TEXT PK | Migration version identifier |
| `applied_at` | TEXT NOT NULL | When the migration was applied |

## Entry Types (Taxonomy)

| Type | Description |
|------|-------------|
| `log` | General activity log |
| `idea` | Creative idea or concept |
| `insight` | Learning or realization |
| `candidate_task` | Something that needs to be done |
| `reference` | Link, article, or resource |
| `decision` | A decision that was made |
| `problem` | Bug, error, or blocker |
| `review` | Review or retrospective |
| `general_note` | Uncategorized note |

## Confidence Rules

- `confidence >= 0.70`: Entry is marked as `valid`.
- `confidence < 0.70`: Entry is marked as `needs_review`, and an audit event is created.

## Timezone Rules

- Stored timestamps remain UTC ISO 8601 values.
- `pie report daily --date YYYY-MM-DD` interprets `YYYY-MM-DD` as a local calendar day.
- The default local timezone is `America/Sao_Paulo`.
- Override it with `PIE_LOCAL_TIMEZONE` using an IANA timezone name, for example `UTC` or `America/New_York`.
- Daily reports convert the local day boundary to UTC before selecting structured entries.
